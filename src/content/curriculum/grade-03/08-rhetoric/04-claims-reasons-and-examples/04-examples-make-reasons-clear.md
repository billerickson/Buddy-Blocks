## Teaching Goal

Show how a specific example or detail can strengthen a reason.

## Student Outcome

The student can add or choose an example that clarifies a reason for a claim.

## Misconception Checks

A second reason is always an example; an example can be vague; examples do not need to connect to the reason.

## Teaching Note

Put the claim, reason, and example on three linked cards and ask what job each one does.

## Questions

```question
key: u04_l04_q01_strongest_example
type: multiple-choice
prompt: "Claim: The class helper job teaches responsibility. Reason: The helper must remember important tasks. Which example is strongest?"
choices:
  - For example, the helper checks the homework bin before lunch.
  - For example, helpers are people.
  - For example, the job has a name.
  - For example, responsibility is important.
correctAnswer: For example, the helper checks the homework bin before lunch.
explanation: The example shows a specific task the helper must remember.
hint: An example should show the reason happening.
```

```question
key: u04_l04_q03_reason_to_example
type: match-pairs
prompt: Match each reason to the example that clarifies it.
pairs:
  - left: The rule keeps the line safe.
    right: Students leave space instead of pushing.
  - left: The chart helps us remember jobs.
    right: It shows who feeds the class plant each day.
  - left: The map makes the route clearer.
    right: It marks the gym with a star.
  - left: The quiet signal saves time.
    right: Students stop talking without another reminder.
explanation: A clarifying example shows the reason in a specific case.
hint: Match each reason with the example that shows how it works.
```

  For example, we can check off 'add title' before we turn in the poster.

  - It shows how the checklist helps the group remember a step.
  - It changes the claim to a new topic.
  - It describes the color of the poster.
  - It asks a question about the group.

## Teaching Goal

Combine claim, reason, and example into a short supported response with a simple conclusion.

## Student Outcome

The student can write or complete a compact response that includes a claim, one reason, one example, and a closing sentence.

## Misconception Checks

Longer is automatically stronger; the conclusion should introduce a new claim; examples can replace reasons.

## Teaching Note

Model a four-sentence response and color-code each part by job.

## Questions

```question
key: u04_l05_q01_order_response
type: order-items
prompt: Put the supported response in the best order.
items:
  - For example, everyone could see the homework date.
  - The board should show homework reminders.
  - This would help students come prepared.
  - Reminders help students remember assignments.
correctOrder:
  - The board should show homework reminders.
  - Reminders help students remember assignments.
  - For example, everyone could see the homework date.
  - This would help students come prepared.
explanation: A clear response moves from claim to reason to example to conclusion.
hint: Start with what the writer thinks should happen.
```

```question
key: u04_l05_q02_complete_supported_response
type: multi-blank-cloze
prompt: Complete the short response.
parts:
  - "Our class should keep extra pencils near the door because "
  - ". For example, "
  - .
blanks:
  - correctAnswer: students sometimes forget one
    choices:
      - students sometimes forget one
      - pencils can be sharpened
      - the door is near the hallway
  - correctAnswer: a student could borrow one before the quiz starts
    choices:
      - a student could borrow one before the quiz starts
      - the pencil box could sit near the door
      - the pencils could be different lengths
explanation: The reason and example both support keeping extra pencils nearby.
hint: The first blank should tell why; the second should show one specific case.
```

```question
key: u04_l05_q03_best_conclusion
type: multiple-choice
prompt: "Claim: The class should use a project checklist. Reason: It helps groups remember each step. Example: Groups can check off 'write names' before turning work in. Which conclusion fits best?"
choices:
  - A checklist would help groups finish carefully.
  - Markers come in many colors.
  - Some projects are about science.
  - Names can be short or long.
correctAnswer: A checklist would help groups finish carefully.
explanation: The conclusion returns to the claim and support without adding a new topic.
hint: A conclusion should close the same idea.
```

```question
key: u04_l05_q04_missing_part
type: passage-question
prompt: Read the response and answer.
passageTitle: Desk Claim
passage: |
  Students should clean their desks every Friday.
  For example, Mia found her math folder faster after cleaning.
  This would make Monday easier.
question: Which part is missing?
choices:
  - a reason explaining why Friday cleaning helps
  - a claim
  - an example
  - a conclusion
correctAnswer: a reason explaining why Friday cleaning helps
explanation: The response has a claim, example, and conclusion, but it needs a clear because reason that explains why Friday cleaning helps.
hint: Look for the part that answers why.
```

```question
key: u04_l05_q05_write_supported_response
type: multi-blank-cloze
prompt: Complete the short response with a reason and example.
parts:
  - "The class should use a quiet signal because "
  - ". For example, "
  - .
blanks:
  - correctAnswer: it saves time
    choices:
      - it saves time
      - signals can have names
      - the room has a clock
  - correctAnswer: students can stop talking without repeated reminders
    choices:
      - students can stop talking without repeated reminders
      - students can choose a favorite color
      - the signal could be written on paper
explanation: The reason explains why the quiet signal helps, and the example shows one specific case.
hint: The first blank should answer why; the second should show the reason happening.
```

```question
key: u04_l05_q06_part_to_job
type: match-pairs
prompt: Match each response part to its job.
pairs:
  - left: claim
    right: says what the writer thinks
  - left: reason
    right: answers why the claim makes sense
  - left: example
    right: shows the reason in a specific case
  - left: conclusion
    right: closes the response by returning to the idea
explanation: Each part has a different job in a supported response.
hint: Think about what each part does for the audience.
```

## Teaching Goal

Improve support by replacing vague, weak, or irrelevant reasons and examples.

## Student Outcome

The student can choose a revision that strengthens support for a claim or evaluation.

## Misconception Checks

Vague support is fine if the claim is clear; adding any example improves support; repeating the claim strengthens it.

## Teaching Note

Use a response with "because it is good" and revise it into a reason plus example.

## Questions

```question
key: u06_l03_q01_best_support_revision
type: multiple-choice
prompt: "Draft: 'The class should use a timer because it is good.' Which revision strengthens the reason?"
choices:
  - because it helps groups know when to switch turns
  - because it is helpful
  - because the timer has a label
  - because the timer is kept on the shelf
correctAnswer: because it helps groups know when to switch turns
explanation: The revision gives a specific reason the timer helps.
hint: Choose the reason that explains why the claim makes sense.
```

```question
key: u06_l03_q02_revise_vague_reason
type: multiple-choice
prompt: Which revision gives a stronger reason?
choices:
  - We should make a supply list because it helps us remember what to bring.
  - We should make a supply list because it is helpful.
  - We should make a supply list because it is nice.
  - We should make a supply list because lists are lists.
correctAnswer: We should make a supply list because it helps us remember what to bring.
explanation: The revision explains how the supply list helps.
hint: Tell what the list helps people do.
```

```question
key: u06_l03_q03_weak_support_context
type: passage-question
prompt: Read the draft and answer.
passageTitle: Weak Support
passage: |
  We should practice our presentation again.
  Practice happens in the music room.
question: Why is the support weak?
choices:
  - It does not explain why more practice would help.
  - It has too many examples.
  - It is too respectful.
  - It comes after the claim.
correctAnswer: It does not explain why more practice would help.
explanation: Where practice happens does not explain why more practice would help.
hint: Ask whether the support answers why.
```

```question
key: u06_l03_q04_weakness_to_fix
type: match-pairs
prompt: Match each weak support problem to the best fix.
pairs:
  - left: The reason is vague.
    right: Tell exactly how it helps.
  - left: The example changes the topic.
    right: Choose an example about the claim.
  - left: The claim is repeated.
    right: Add a new why statement.
  - left: The support is missing.
    right: Add one reason before the example.
explanation: Different support problems need different revision moves.
hint: Match the weakness with the action that would repair it.
```

```question
key: u06_l03_q05_add_specific_example
type: multiple-choice
prompt: "Draft: 'The class should use name tags because they help visitors.' Which added example is most specific?"

  - For example, a visitor could greet each student by name.
  - For example, name tags are helpful.
  - For example, visitors visit places.
  - For example, the tags could be on a table.

explanation: The example shows one specific way name tags help visitors.

```

```question
key: u06_l03_q06_repetition_not_revision
type: multiple-choice
prompt: Which revision does NOT really strengthen the support?
choices:
  - The class should use a checklist because a checklist is a list.
  - The class should use a checklist because it reminds us of each step.
  - The class should use a checklist because groups can mark completed tasks.
  - The class should use a checklist because it helps prevent forgotten steps.
correctAnswer: The class should use a checklist because a checklist is a list.
explanation: This repeats the word instead of giving a reason.
hint: A stronger reason tells why the claim makes sense.
```
